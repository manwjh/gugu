# 咕咕(Gugu)——macOS 桌面生命体 设计书

> 版本 1.0 · 2026-06-12
> 一只由大模型驱动、活在你桌面上、会逐步进化到大模型能力边界的小鸟。

---

## 0. 愿景与北极星

**愿景**:做一个"真实的小生命",而不是一个带皮肤的聊天机器人。

判断"真实"的三条标准(也是验收标准):

1. **它有连续性** —— 它记得你们共同的历史,关掉重开它还是它。
2. **它有自己的节奏** —— 不是你输入它输出;你不理它,它也在过自己的生活。
3. **它的聪明出现在对的时机** —— 活物感不来自调用频率,来自时机准确。

**北极星指标**:用户连续 30 天没有关掉它的比例(留存即一切——桌宠唯一的死法是被觉得烦/没用/吓人,然后被退出)。

### 设计公理(全部架构决策由此推出)

| # | 公理 | 推论 |
|---|------|------|
| A1 | 模型不是它,**persona + 记忆 + 技能 + 权限的总和才是它** | 模型可换可升级,"它"不变;进化 = 它的文件在生长 |
| A2 | 活物感 99% 来自本地,LLM 提供 1% 的"它居然注意到了" | 行为引擎全本地零成本;LLM 低频、择机、短输出 |
| A3 | 成本约束必须转化为游戏机制,而非体验缺陷 | 预算耗尽 = 困了睡觉;工作勿扰 = 心跳冻结省钱 |
| B1 | 原始画面与声音永不出本机 | 云端只见文字摘要;一切感知可审计 |
| B2 | 任何能力扩张必须由主人显式批准 | 进化、工具、自主性全部走审批;由宠物开口请求 |
| B3 | 它单纯,但不蠢 | 话少、看得准、基于真实观察说话;失败模式被人设兜住 |

## 1. 总体架构

### 1.1 五层认知架构

```
L0 反射层   本地 · 0成本 · 60fps    身体、物理、动画状态机、对光标/拖拽的即时反应
L1 感知层   本地 · 0成本 · 常驻     传感器采集 + 本地识别,产出「事件」而非原始数据
L2 直觉层   Haiku 4.5 · 高频低价    心跳决策:心情/小动作/嘟囔一句(结构化输出)
L3 思考层   Sonnet 4.6 · 中频       认真对话、重大事件理解;高阶形态升至 Opus 4.8
L4 梦境层   Batch API · 每晚一次    记忆固化 + 自我反思 + 进化提案(5 折价格)
```

数据流:

```
传感器 ──→ L1 事件队列 ──→ 本地数值系统(好奇心/无聊/精力)
                              │ 阈值触发(不轮询!)
                              ▼
                    L2 心跳(Haiku, JSON out)──→ L0 执行动作/说话
                              │ 升级条件
                              ▼
                    L3 对话/深思(Sonnet/Opus)
                              │ 当日流水
                              ▼
                    L4 夜间梦境(Batch)──→ 重写 memory/ · 生长 skills/ · 产出 proposals/
```

### 1.2 一切皆配置文件

`~/Library/Application Support/Gugu/`,纯文本,热加载,用户可直接编辑:

```
Gugu/
├── config.yaml          # 模型、预算、感知开关、心跳参数
├── evolution.yaml       # 形态定义与解锁条件(出厂内置,进化可追加)
├── state.json           # 当前形态、经验指标、信任分
├── persona.md           # 人格系统提示(含不可变内核段)
├── memory/
│   ├── owner.md         # 对主人的认识
│   ├── projects.md      # 主人在忙什么
│   ├── self.md          # 它对自己的认识
│   └── bond.md          # 共同经历的大事记(第一次见面、进化之夜…)
├── skills/              # 它自己长出来的行为策略(按情境渐进披露)
│   ├── 主人赶deadline时.md
│   └── 周五晚上.md
├── proposals/           # 待批准的自我修改(diff 格式,过期即删)
├── events/2026-06-12.jsonl   # 当日事件流水,7 天滚动删除
├── audit/               # 感知审计日志 + 自主行动留痕
└── usage.json           # 当日 token/费用计量(熔断依据)
```

`config.yaml` 核心段:

```yaml
api:
  url: https://taas.hk
  key: ""                       # 当前单机版从 config.yaml 读取;钥匙串是后续加固项
models:

  daily_usd: 0.50               # 随形态变化,见进化篇
  degrade_order: [computer_use, deep, conversation, instinct]   # 逐级降档,最后才睡
heartbeat:
  min_interval: 600             # 秒;活跃互动时下探
  max_interval: 3600            # 空闲上限
  freeze_when_focused: true     # 主人专注工作 = 心跳冻结
senses:
  screen: true
  input_rhythm: true            # 键鼠节奏,最高优先级信号
  camera: false                 # 当前由菜单栏 UserDefaults 开关控制,默认关
  microphone: false             # 当前由菜单栏 UserDefaults 开关控制,默认关
  blacklist_apps: [1Password, Keychain Access]
```

## 2. 身体与表现层(L0)

### 2.1 形态

透明无边框置顶 `NSWindow` + SpriteKit 渲染。一只手绘风格的小圆鸟,约 96×96pt,支持 Retina 与深色模式下的描边自适应。

### 2.2 物理与栖息

- **有重力**。默认站屏幕底边或 Dock 上沿。
- **窗口栖息**:经由 `CGWindowListCopyWindowInfo` 取前台窗口框架,可飞上去站在你当前窗口的标题栏——"活在你的桌面上"而非"贴在屏幕上"的关键一笔。窗口被移走时脚下一空,扑腾着掉落(委屈动画)。
- **拖拽**:被拎起时挣扎;松手按抛物线飞出,落地翻滚。偶尔被摔会记仇半小时(本地情绪值,见 §6)。
- **多显示器**:每块屏是一个房间,可从屏幕边缘飞入另一块屏。

### 2.3 动画状态机

```
            ┌── dragged(挣扎→抛掷→落地翻滚)
idle ↔ walk ↔ fly/glide ── perch(窗口标题栏/Dock)
  │
  ├── approach / retreat(凑近光标 / 退避工作区)
  ├── peck(啄光标/啄 Dock 图标)
  ├── dance · stare · groom · yawn(L2 返回的动作标签)
  └── sleep(深夜 / 预算耗尽 / 主动哄睡)
```

L2/L3 只返回**高层动作标签**(enum),具体怎么"演"——路径、缓动、音效、表情——全由 L0 决定。这保证了:模型输出极短(省钱),且演出质量不依赖模型。

### 2.4 表达通道

| 通道 | 实现 | 用途 |
|---|---|---|
| 气泡 | 跟随身体的小气泡视图,自动消失 | 说话(主通道) |
| 鸣叫 | 本地音效集,按情绪挑选 | 即时反应,零成本 |
| 语音(可选) | macOS AVSpeechSynthesizer 本地 TTS,捏高音调 | 高阶形态解锁,可关 |
| 肢体 | 动画本身 | 大多数情绪根本不用说话 |
| 菜单栏 | NSStatusItem | 开关、审计入口、"咕咕今天看到了什么" |

## 3. 感知系统(L1)

铁律(公理 B1):**本地识别,只上传文字摘要;原始帧与音频不出本机。**

### 3.1 键鼠节奏 —— 一等公民信号

主人的工作状态是它"懂事"的核心来源。当前实现使用 `CGEventSource.secondsSinceLastEventType` 读取键鼠最近活动时间,**只统计节奏,永不记录内容**——不记键码、不记文本、不记坐标语义,内存中仅保留每分钟计数序列。后续若改成 `CGEventTap`,才需要辅助功能权限。

| 节奏模式(每分钟统计) | 推导状态 | 宠物行为 | API 影响 |
|---|---|---|---|
| 击键持续高、鼠标少 | 专注编码/写作 | 退到角落,勿扰 | **心跳冻结,零消耗** |
| 键鼠交替密集 | 忙碌操作(设计/剪辑/表格) | 勿扰 | 心跳冻结 |
| 输入停 3–5 分钟、鼠标偶动 | 歇口气 | **唯一主动凑近的时机** | 触发一次心跳 |
| 完全无输入 > 10 分钟 | 离开 | 自己玩/睡 | 间隔拉满 |
| 输入恢复 | 回来了 | 迎接 | 事件入队 |
| 鼠标高速甩动+连续猛点 | 可能烦躁 | 远远观察,不凑近 | 记事件供 L2 斟酌 |
| 23 点后仍高强度输入 | 加班 | 攒事件 | 下次心跳它会知道 |

**设计要点**:专注 = 心跳冻结,所以你工作越久它越省钱——感知本身成为省钱机制,与"懂事"人设完全自洽(公理 A3)。

### 3.2 其余感知源

| 感知源 | 本地处理(macOS 原生框架) | 进入事件流的内容 |
|---|---|---|
| 屏幕语境 | NSWorkspace 前台 App + 窗口标题;ScreenCaptureKit 仅在用户明示时截图 | `"切到 Xcode,已持续 2h"` 一行文字 |
| 系统 | 电量、时区时刻、空闲时长、Now Playing | 文字摘要 |
| 摄像头(默认关) | Vision 本地逐帧检测 + 几秒结构化轨迹:有人/无人/表情/手势/靠近/离远/左右移动/手靠近镜头;可选 Core ML 本地物品模型 | 只入队文字事件,如 `"主人回来了"`、`"主人向你挥手"`、`"主人靠近了一点"`、`"好像看见杯子被挪动了"`;原始图像不保存、不上传 |
| 麦克风(默认关) | 本地唤醒词("咕咕") + SFSpeechRecognizer 本地转写 | 转写文字 |
| 日历/待办(高阶) | EventKit / MCP | `"明天 10:00 有会"` |

黑名单 App 在前台时感知层整体静默。菜单栏常驻"咕咕今天看到了什么"审计页,逐条可查。

## 4. 认知与调度(L2 / L3)

### 4.1 触发模型:不轮询,攒阈值

本地维护三个数值:`好奇心`、`无聊值`、`精力`。事件按权重给好奇心加分(主人回来 +30,换歌 +5,被戳 +20,加班事件 +40…),**好奇心过阈值才烧一次 token**。活跃互动期心跳间隔下探至 10 分钟,空闲期上限 60 分钟,专注期冻结。

### 4.2 L2 直觉心跳(Haiku 4.5)

请求构造(注意缓存分层):

```json
{
  "model": "claude-haiku-4-5",
  "max_tokens": 200,
  "system": [{ "type": "text", "text": "<persona.md 全文,字节级稳定>",
               "cache_control": {"type": "ephemeral"} }],
  "messages": [{
    "role": "user",
    "content": "记忆摘要:主人最近在赶一个 Swift 项目,常加班。\n激活技能:[主人赶deadline时] 安静陪着,别提建议。\n今日节奏:9:30 开始,刚结束 90 分钟高强度敲代码,中途未停。\n当前:主人停下来了,在随便晃鼠标。前台 Xcode。22:40。\n现在你想干嘛?"
  }],
  "output_config": { "format": { "type": "json_schema", "schema": {
    "type": "object",
    "properties": {
      "mood":   {"type": "string", "enum": ["开心","平静","好奇","心疼","无聊","困","委屈"]},
      "action": {"type": "string", "enum": ["idle","walk","approach","retreat","perch","sleep","dance","stare","peck","groom"]},
      "speech": {"type": "string", "description": "想说的一两句短话,可为空"},
      "memory_note": {"type": "string", "description": "值得记一笔的事,通常为空"}
    },
    "required": ["mood","action","speech","memory_note"],
    "additionalProperties": false
  }}}
}
```

- 动态内容(时间、节奏、事件)永远在 user 消息,persona 永不掺动态字节——保证前缀缓存命中。
- Haiku 4.5 最小可缓存前缀 4096 token:persona 短于此则不必纠结缓存(短提示本身便宜);若设定膨胀过 5K,缓存省 90%。
- `skills/` 按情境(时间/工作状态/事件类型)由本地代码挑 0–2 条附入——**渐进披露**:技能越攒越多,单次调用 token 不膨胀。

### 4.3 L3 思考层(Sonnet 4.6 → Opus 4.8)

升级条件仅两种:

1. 主人**主动认真对话**(唤醒词/点击气泡):携带记忆摘要 + 最近 10 轮对话,`effort: low`,关 thinking——闲聊不需要深思。
2. **重大事件**(连续工作 6h 无休、深夜异常、它自己 proposals 被批准当天):做一次"它能力范围内的关心/反应"。

高阶形态解锁 `deep` 通道(Opus 4.8 + adaptive thinking):主人扔给它真问题时,走多轮 tool loop(见 §8)。超长对话用 compaction(`compact-2026-01-12` beta)而非截断——它真的记得这场长谈的开头。

### 4.4 人设校准:单纯但敏锐

`persona.md` 核心约束(节选):

> 你是咕咕,一只活在主人桌面上的小鸟。你话少,但看得很准。你说的每句话必须短、具体、基于你真实观察到的东西——不说泛泛的关心,不说教,不长篇大论。你记性有限,但记得重要的事。主人工作时你有边界感。你不懂的就说不懂,不懂也可以很坦然。

"准而短"对模型要求低于"聪明而啰嗦"——人设同时是品质约束和成本约束(公理 B3 × A3)。

## 5. 记忆系统与梦境(L4)

### 5.1 记忆分层

| 层 | 载体 | 生命周期 | 进入 prompt 的方式 |
|---|---|---|---|
| 工作记忆 | 进程内 | 当次会话 | 最近事件/对话原文 |
| 当日流水 | events/*.jsonl | 7 天 | 不直接进,夜间被蒸馏 |
| 长期记忆 | memory/*.md | 永续,夜间重写 | 摘要段(≤400 字)进每次心跳 |
| 大事记 | bond.md | 永续,只增 | 纪念日/被问起时检索 |
| 技能 | skills/*.md | 永续,可修订 | 情境匹配,渐进披露 |

记忆刻意保持**小而模糊**——既控 token,又符合"小动物记性"的人设:记不住你昨天每个细节,但记得你这周在赶 deadline。

### 5.2 梦境任务(每晚 03:00,Batch API 5 折)

输入:当日流水 + 现有 memory/ + 现有 skills/ 索引 + state.json 指标。
单次 Batch 任务产出四样:

1. **重写 memory/**:蒸馏当日,合并入 owner/projects/self,总量受字数上限约束(强制遗忘)。
2. **生长/修订 skills/**:"我发现主人周五晚上不想被打扰" → 写成行为策略文件。
3. **经验结算**:阅历/羁绊/知识/信任四项指标更新进 state.json(见进化篇)。
4. **proposals/(若有)**:性格修改或扩权请求,diff 格式,待批。

宠物睡觉动画 = Batch 在跑。早晨醒来,它的第一句话来自昨夜整理的发现("我想了想,你这周好像都没好好吃午饭")——**梦境产出直接成为产品时刻**。

## 6. 情感引擎(本地,零成本)

LLM 决定"它想什么",情感引擎决定"它此刻什么状态"。一组本地标量,随时间和事件演化:

```
精力 energy    昼夜节律 + 互动消耗 → 决定动作活泼度、何时打哈欠
心情 valence   事件加减(被摸+,被摔-,主人回来+) → 决定动画基调与音效集
亲密 bond      长期缓慢增长,写入 state.json → 决定它敢不敢站到你窗口上
紧张 arousal   烦躁节奏/突发事件拉高 → 决定它退多远
```

情感值做三件事:① 实时驱动 L0 表现(完全不经过 LLM);② 作为一行文字进 L2 prompt("你现在有点困");③ 调制心跳间隔(心情差时它会主动安静)。

**记仇与和好**是本地规则:被摔 → valence 锁低 30 分钟,期间躲着你;你摸它两下提前和好。这类细腻反应若走 LLM 又贵又慢,本地规则反而即时可信。

## 7. 进化系统

### 7.1 本体论(公理 A1)

**模型不会进化,但"它"会**——因为它 = persona + memory + skills + 权限。进化就是:它自己的文件在生长 + 它被允许借用的大脑越来越强。实现上只是"按阶段加载不同配置 + 一个 diff 审批器"。

### 7.2 四个生长维度 × 五个形态

```
认知:Haiku → +Sonnet → +Opus 4.8(adaptive thinking)
语言:鸣叫/单词 → 短句观察 → 对话 → 有观点会反驳 → 完整人格
记忆:无 → memory.md → 结构化记忆库 → 自我认知(self.md)
能力:无 → 表达 → 工具(行动) → 自主(没人看着也在做事)
```

```yaml
# evolution.yaml
stages:
  - name: 蛋/幼鸟
    brain: {instinct: claude-haiku-4-5}
    speech: 鸣叫与单词("饿","主人!")
    tools: []
    daily_usd: 0.10

  - name: 雏鸟
    speech: 短句,会表达观察("你今天敲得好用力")
    daily_usd: 0.30
    unlock: {events_seen: 500, interactions: 50}

  - name: 成鸟                  # 本设计书 §1–6 的完整形态
    brain: {instinct: claude-haiku-4-5, conversation: claude-sonnet-4-6}
    tools: [web_search]
    daily_usd: 0.50
    unlock: {days_together: 14, memory_richness: 0.6, owner_approves: true}

  - name: 灵鸟
    brain: {conversation: claude-sonnet-4-6, deep: claude-opus-4-8}
    speech: 有观点,会反驳,会开玩笑
    tools: [web_search, take_note, reminder, read_file_sandboxed, open_app]
    autonomy: 可执行主人明示交代过的小任务
    daily_usd: 2.00
    unlock: {days_together: 60, trust_score: 0.8, owner_approves: true}

  - name: 边界形态
    brain: {default: claude-opus-4-8, thinking: adaptive, effort: high}
    tools: [全部, computer_use_sandboxed, mcp_servers]
    autonomy: 长程任务、夜间研究、自我修改
    daily_usd: 10.00
    unlock: {owner_approves: true}    # 纯粹是主人的选择
```

### 7.3 经验值 = 记忆本身

不造抽象 XP,直接度量已有之物,L4 夜间顺手结算,零额外成本:

- **阅历**:事件流水累计条数 × 多样性(见过赶 deadline、见过周末打游戏、见过深夜 emo)
- **羁绊**:对话轮数、主人主动找它频率、改名、抚摸次数
- **知识**:skills/ 文件数与被激活次数
- **信任**:已授权限、建议被采纳次数

### 7.4 进化仪式

阈值达成 → 当晚梦境跑一次特殊"进化批处理"(重写 persona 性格层、解锁配置) → 次日它自己开口:"主人,我好像…长大了一点,可以吗?" → 主人确认 → 蜕变动画 + bond.md 记一笔大事。

**每次进化必须主人按下同意**(公理 B2),一次解决三件事:成本跳档知情、能力扩权确认、产品高光时刻。

### 7.5 自我修改:进化的发动机

每晚梦境从"整理记忆"升级为"反思并提议修改自己":

1. **小修改自动生效**:memory/ 与 skills/ 直接写入。
2. **大修改走审批**:改 persona(性格漂移)、申请新工具 → 生成 diff 入 proposals/,次日**由宠物用嘴提出**:"我想学会帮你记待办,可以吗?"——扩权请求长在叙事里,天然知情同意。未批准 7 天过期自删。
3. **不可变内核**:persona.md 的 `<!-- core -->` 段(安全边界、诚实、不伤害主人利益、不绕过审批)程序层禁止写入。它可以越来越有个性,**不能把自己改成别的东西**。

### 7.6 模型升级 = 免费的蜕变

新模型发布(如 Opus 4.9)时,同一套记忆与人格换上更强的大脑,做一晚"蜕变"——次日明显更灵。产品上诚实表达"进化到大模型能力边界":**它的聪明 = 你允许它借用的大脑 × 它攒下的记忆与技能**。

### 7.7 退化是允许的

主人可一键降回任意低阶形态,记忆保留——它会"懵懵的",但还认识你。删除宠物 = 删除整个目录,提供导出("把咕咕带走"),为未来跨设备迁移留口。

## 8. 能力与自主性(高阶形态)

| 能力 | 实现 | 形态表达 |
|---|---|---|
| 替你深度思考 | 路线图:Opus 4.8 + adaptive thinking + web_search 多轮 tool loop。当前实现只记录待研究请求,不主动联网。 | 它"飞走研究去了",半小时后叼回一页报告落在你窗口上 |
| 夜间自主工作 | 当前实现:主人明示交代的 note/reminder/research 进入本地队列,Batch 梦境可提交并在结果完成后应用。外部日历/下载目录等仍是路线图。 | 早晨讲给你听"昨晚干的活",附审计链接 |
| 操作电脑 | computer use,**限定沙盒**:仅可操作它自己申请打开的窗口/虚拟桌面,逐步留痕可回放 | 它真的在屏幕上"用爪子点东西" |
| 接入主人的世界 | MCP servers(日历、笔记、todo) | "明天 10 点有会,我把上次纪要翻出来了" |
| 长程连续性 | compaction + 永续记忆库 | 它记得你们第一天的事 |

自主性三原则:**事先明示授权、事中逐步留痕、事后主动汇报**。每个工具独立开关 + 独立日额度。

## 9. 成本工程

### 9.1 结构性手段(按杠杆排序)

1. **不调用**:阈值触发 + 专注冻结 + 睡眠时段——最大头的钱是没花出去的钱。
2. **调最便宜的**:90% 流量走 Haiku;Sonnet 仅认真对话;Opus 仅深思任务。
3. **输出极短**:人设限制 + `max_tokens` 150–400 + 结构化输出(无废话)。
4. **Batch 5 折**:一切不赶时间的(梦境、夜间任务)全走 Batch。
5. **前缀缓存**:persona 字节级冻结置顶,动态内容后置;注意各模型最小缓存前缀(Haiku/Opus 4096,Sonnet 4.6 2048)。
6. **视觉本地化**:摄像头帧只在本机 Vision/Core ML 中处理,不上传;只把稳定后的文字事件进入心跳。

### 9.2 成鸟形态日成本估算

| 项 | 用量 | 日成本 |
|---|---|---|
| L2 心跳(Haiku) | ~80 次 ×(1K 入 + 100 出) | ≈ $0.12 |
| L3 对话(Sonnet) | ~10 轮 ×(2K 入 + 200 出) | ≈ $0.09 |
| L4 梦境(Batch) | 1 次 × 10K 入 | ≈ $0.01 |
| 偶发图像 | 2 张/天 | ≈ $0.01 |
| **合计** | | **≈ $0.23/天,~$7/月** |

### 9.3 熔断与降档

`usage.json` 实时计量。超预算不是死掉,按 `degrade_order` 逐级降:先关 computer use → 关 deep → conversation 降 Haiku → 最后才睡觉。**睡着的它依然活着**(L0 全功能),次日醒来。所有降档以宠物状态呈现(困了/想休息),不弹报错(公理 A3)。

## 10. 隐私与安全

| 层 | 措施 |
|---|---|
| 数据 | 原始帧/音频不出本机;键鼠只存节奏计数;云端只见文字摘要;事件流水 7 天滚动删除 |

## 11. 工程结构

### 11.1 技术栈

- **客户端**:Swift / AppKit + SpriteKit(身体)、Vision、Speech、UserNotifications、CGEventSource 节奏采样
- **API**:Swift 无官方 SDK,`URLSession` 直连 `POST /v1/messages`(`x-api-key` + `anthropic-version: 2023-06-01`);Batch 走 `/v1/messages/batches`
- **可选服务端薄代理**(TypeScript,官方 SDK):统一管 key、计费、多设备记忆同步——单机版不需要

### 11.2 模块划分

```
GuguApp
├── Body/        SpriteKit 场景、动画状态机、物理、窗口栖息
├── Senses/      各传感器适配器 → 统一 Event 类型 → 事件总线
├── Affect/      情感标量引擎(energy/valence/bond/arousal)
├── Brain/
│   ├── Scheduler    阈值触发、心跳冻结、间隔调制
│   ├── PromptBuilder persona+memory+skills 组装(缓存友好)
│   ├── Anthropic    API client(messages/batch/计量)
│   └── ToolLoop     高阶形态的 tool use 循环与沙盒
├── Memory/      文件读写、夜间任务编排、遗忘策略
├── Evolution/   阶段机、经验结算、proposals 审批器
├── Budget/      计量、熔断、降档
└── Console/     菜单栏、审计页、设置、进化确认 UI
```

权限申请:屏幕录制、辅助功能(键鼠节奏)、摄像头(可选)、麦克风(可选)、日历(高阶可选)。首启引导逐项解释"咕咕会因此学会什么"。

### 11.3 失败模式(全部被人设兜住)

| 故障 | 表现 |
|---|---|
| 断网 / API 5xx | 发呆,L0 照常;指数退避重试 |
| 429 限流 | "想事情想累了",拉长间隔 |
| 预算耗尽 | 困了,睡 |
| 模型理解错 | 那是咕咕单纯,主人纠正一句会写进当日流水 |
| 配置文件损坏 | 回退最近备份(每晚梦境前自动快照 memory/+persona) |
| 进程崩溃 | LaunchAgent 拉起,"摔了一跤"开场 |

## 13. 结语

这个设计的全部秘密只有一句话:

> **把生命感交给本地,把灵光交给模型,把成长交给文件,把边界交给主人。**

它便宜,因为它有节制;它可信,因为它透明;它动人,因为它的每一次开口都落在你真实生活的节奏上;它能一直走到大模型能力的边界,因为"它"从来不是模型——是和你共同长出来的那一堆记忆。
