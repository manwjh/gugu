# FAQ

## 这是成品 App 吗?

现在是 source-first release,主要给愿意从源码运行的 macOS 用户和开发者。还没有正式 Developer ID 签名、公证、自动更新和下载即用安装包。

## 为什么需要 API key?

咕咕的 L0/L1 身体和本地感知不需要模型,但 L2/L3/L4 的心跳、对话和梦境整理需要 OpenAI Chat Completions 兼容模型接口。你需要在本机 `config.yaml` 里配置自己的 endpoint 和 key。

## 会记录我输入了什么吗?

不会。键鼠节奏感知只统计活动节奏,不记录按键内容。

## 摄像头和麦克风默认会开吗?

不会。摄像头、麦克风、TTS 都默认关闭,需要你在菜单栏手动开启。开启后原始画面和音频也不上传、不保存。

## 数据在哪里?

默认在:

```text
~/Library/Application Support/Gugu/
```

里面包括配置、记忆、事件、审计、提案、快照和 token 用量。大多数文件是纯文本或 JSONL。

## 现在适合什么人试?

- 愿意从源码运行 macOS app 的用户
- 对 AI agent、桌宠、长期记忆、本地隐私边界感兴趣的开发者
- 能接受早期项目安装和配置摩擦的人

## 现在不适合什么人?

- 只想下载一个已签名、公证、自动更新的普通 Mac app 的用户
- 不想配置模型 endpoint/API key 的用户
- 需要企业级隐私合规文档的用户
